01 · Preprocessing

Feature Normalisation

Min-Max Normalisation → [−1, 1]
xnorm = 2⋅ x−xmin xmax−xmin − 1
Variables
xRaw input feature value (e.g. Age = 54)
x_minMinimum value of that feature in the full dataset
x_maxMaximum value of that feature in the full dataset
x_normScaled output — always lies in the range [−1, 1]
Applied to all 6 numerical features: Age, RestingBP, Cholesterol, FastingBS, MaxHR, Oldpeak.
One-Hot Encoding
yoh = [ δ0,c , δ1,c ] where δi,c = { 1 ifi=c 0 otherwise
Variables
cTrue class label (0 = no disease, 1 = heart disease)
iIndex being tested (0 or 1)
δ_{i,c}Kronecker delta — equals 1 only when i matches c
y_ohOne-hot vector: [1, 0] for class 0; [0, 1] for class 1
Label 0 → [1, 0]  ·  Label 1 → [0, 1]
02 · Forward Pass

Layer Computations

Pre-activation (each layer)
z(l) = a(l−1) W(l) + b(l)
Variables
lLayer index: 1 = Hidden1, 2 = Hidden2, 3 = Output
z^(l)Pre-activation (raw weighted sum) at layer l
a^(l−1)Activated output from the previous layer; equals X when l = 1
W^(l)Weight matrix connecting layer l−1 to layer l
b^(l)Bias vector added at layer l (shape: 1 × neurons in l)
z¹ = X·W¹ + b¹  (6→3)  ·  z² = a¹·W² + b²  (3→4)  ·  z³ = a²·W³ + b³  (4→2)
03 · Activation Functions

Hidden Layer Activations

ReLU (Rectified Linear Unit)
ReLU(z) = max(0,z) = { z ifz>0 0 otherwise
Variables
zPre-activation scalar (one element of z^(l))
ReLU(z)Activated output — negative values clamped to zero
ReLU′(z)Derivative used in backprop: 1 if z > 0, else 0
max(0, z)Equivalent expression — take the larger of 0 and z
Defined in the code but not used for training — Leaky ReLU is used instead to avoid dead neurons.
Leaky ReLU — used in training
LeakyReLU(z) = { z ifz>0 αz otherwise α=0.01
Variables
zPre-activation value passed into the function
α (alpha)Negative-slope coefficient — fixed at 0.01 in this model
αzSmall scaled output for negative inputs — keeps gradient alive
LeakyReLU′Derivative: 1 if z > 0, else α = 0.01
Prevents dead neurons — unlike ReLU, negative inputs still produce a non-zero gradient of 0.01.
04 · Output Layer

Softmax

Softmax — converts logits to probabilities
softmax (z)i = ezi ∑ j=1 K ezj
Variables
zLogit vector from the output layer z³, shape [N, 2]
z_iLogit for the class i we want the probability for
z_jLogit for class j — summed over all K classes in the denominator
KTotal number of classes — K = 2 here
e^z_iExponentiated logit — ensures all values are positive before dividing
Numerically stabilised: e^(z − max(z)) is computed instead of e^z to prevent overflow.
05 · Loss Function

Cross-Entropy Loss

Categorical Cross-Entropy
L = − 1N ∑ n=1 N ∑ k=1 K yn,k ⋅ log ( y^n,k + 10−9 )
Variables
LScalar loss value — the quantity minimised during training
NNumber of training samples (batch size)
KNumber of output classes — K = 2
y_{n,k}One-hot ground truth: 1 for the true class, 0 for the other
ŷ_{n,k}Softmax predicted probability for sample n, class k
10⁻⁹Small epsilon added inside log to prevent log(0) = −∞
Since y is one-hot, only the term where y_{n,k} = 1 contributes — reducing to −log(ŷ of the true class).
06 · Backpropagation

Gradient Computation

Output Layer Error (Softmax + Cross-Entropy combined)
δ(3) = a(3) − Y
Variables
δ^(3)Error signal at the output layer, shape [N, 2]
a^(3)Softmax output probabilities (ŷ) from the final layer
YOne-hot ground truth matrix, shape [N, 2]
a^(3) − YElement-wise difference — how far predictions are from truth
The combined softmax + cross-entropy derivative simplifies elegantly — no chain rule needed at the output.
Hidden Layer Error (Chain Rule)
δ(l) = ( δ(l+1) W(l+1) T ) ∘ f′ ( z(l) )
Variables
δ^(l)Error signal propagated back to layer l
δ^(l+1)Error signal already computed for the next layer ahead
W^(l+1)ᵀTranspose of the weight matrix from the layer ahead
∘Element-wise (Hadamard) multiplication — not matrix multiply
f′(z^(l))Leaky ReLU derivative applied element-wise to z^(l)
z^(l)Pre-activation values saved during the forward pass
Flows error signals layer-by-layer from output toward input, applying the chain rule at each step.
Weight & Bias Gradients
∂L ∂W(l) = 1N a(l−1) T δ(l) ∂L ∂b(l) = 1N ∑ n=1 N δ(l)
Variables
∂L/∂W^(l)Gradient of loss w.r.t. weights at layer l
∂L/∂b^(l)Gradient of loss w.r.t. bias vector at layer l
a^(l−1)ᵀTranspose of previous layer's activations
δ^(l)Error signal at layer l from the chain rule above
1/NBatch-averaging factor — gradients are mean over all samples
In code: dW = a_prev.T @ delta / N and db = delta.mean(axis=0, keepdims=True).
07 · Optimisation

Gradient Descent Update

Weight Update Rule (lr = 0.1, epochs = 5000)
W(l) ← W(l) − η ∂L ∂W(l) b(l) ← b(l) − η ∂L ∂b(l)
Variables
W^(l)Weight matrix at layer l — overwritten in-place each epoch
b^(l)Bias vector at layer l — overwritten in-place each epoch
η (eta)Learning rate — fixed at 0.1; controls the step size
∂L/∂W^(l)Gradient pointing in direction of steepest loss increase
←In-place assignment — subtracts the scaled gradient from current weight
η · ∂L/∂WThe step taken — small enough (η=0.1) to descend gradually
Weights initialised with N(0, 0.5²); biases start at zero. Runs for 5000 epochs total.

Prediction

Argmax Classification
c^ = argmax k softmax ( f ( x ; θ ) ) ⋅ k
Variables
ĉPredicted class — 0 (no disease) or 1 (heart disease)
kClass index iterated over (0 or 1)
argmax_kReturns the index k with the highest probability value
xNormalised input feature vector for a single patient
θ (theta)All learned parameters: W¹, b¹, W², b², W³, b³
f(x; θ)Full forward pass through all 3 layers, producing logits z³
In code: np.argmax(softmax(a2 @ W3 + b3), axis=1).
